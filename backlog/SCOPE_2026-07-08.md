## 2. New issues (#19–#23) — triage summary

- **#23 — index.php fresh-install crash + direct login (Bug).** Fresh-install home page whitescreens after the sidebar loads; litter/wean + colony-stats tables don't populate; direct login on the home page never converts to a logged-in state; sidebar stuck expanded with no toggle.
  **Root cause (grounded):** `index.php` L106–109 calls `$result = $conn->query("call get_colonystats()")` then `$result->fetch_array()` **with no guard**. Under the Phase F `MYSQLI_REPORT_OFF` decision (§7 of the handoff), a missing proc / failed query returns `false` silently, and `false->fetch_array()` fatals → whitescreen. The weantable query (L161) shares the shape on empty results. Direct-login non-conversion is partly by-design under Phase F (single login origin = `databases.php`); the author's proposed fix — redirect to `databases.php` on reconnect from a dead session — is Phase-F-consistent.

- **#22 — cage_location.php crash on empty location (Bug).** Fresh DB whitescreens; selecting a location with no cages whitescreens; `cage_list` select box populates empty. **Same unguarded-result / empty-`IN()` family as #23.**

- **#21 — Manage Cages multi-select move (Feature).** Allow selecting a *subset* of mice in `animal_selection` to move collectively (today: one-at-a-time or all-filtered). Lives in the same `manage_cages.php` staging code as #8 V1 and the 2b `temp_cage` concatenation.

- **#20 — Quick Sac tool (Feature).** Batch-set a common DOD/comment on a set of selected animals. Write-gated bulk mutation; reuses the multi-select selection UX established by #21.

- **#19 — Userbook resources & pages (Feature, large).** User/admin management surface built on Phase F auth: (a) admin provision/revoke DB access, (b) invite/registration flow + email, (c) password reset/recovery, (d) user preferences, (e) admin DB spin-up/connect. A full phase of its own ("Phase G"); decomposes into the five sub-features above.
